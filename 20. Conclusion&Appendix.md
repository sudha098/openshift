## Demo â€“ GitLab Setup

This appendix provides a walkthrough for deploying GitLab on OpenShift for use in CI/CD workflows.

### **Steps Overview**

1. **Create a new project**

   ```sh
   oc new-project gitlab
   ```
2. **Deploy GitLab using Helm Chart or template**

   * Configure Persistent Volumes for data, logs, configs
   * Use LoadBalancer/Route for external access
3. **Configure GitLab Runner**

   ```sh
   oc apply -f gitlab-runner.yaml
   ```
4. **Register the runner**

   * Retrieve GitLab registration token
   * Run registration job or configure via ConfigMap
5. **Verify Runner status** in GitLab under *Settings â†’ CI/CD â†’ Runners*
6. **Create a sample project** and push your repository
7. **Configure `.gitlab-ci.yml`** for CI/CD pipeline execution on OpenShift

### **Outcome**

A fully functional GitLab instance integrated with OpenShift for pipeline execution.

## Pre-Requisite â€“ CI/CD Concepts

Before building CI/CD pipelines with OpenShift or GitLab, understanding core CI/CD principles is essential.

### **Key Concepts**

* **CI (Continuous Integration)**

  * Automated code testing
  * Merge validation
  * Builds triggered on commits

* **CD (Continuous Delivery / Continuous Deployment)**

  * Automated deployment workflow
  * Promotion of applications between environments
  * Versioned artifacts and automated rollouts

### **Pipeline Components**

* Source Code
* Runner/Agent
* Stages (Build, Test, Deploy)
* Artifacts
* Environments (Dev/QA/Prod)
* Approvals (manual or automated)

### **CI/CD Flow Example**

1. Developer pushes code â†’ triggers pipeline
2. Build container image
3. Run tests
4. Push image to registry
5. Deploy to OpenShift via pipeline
6. Optional manual approval for production

### **Tools commonly paired with OpenShift**

* GitLab
* Jenkins
* Tekton Pipelines (OpenShift Pipelines)
* Argo CD (OpenShift GitOps)

## Pre-Requisite â€“ SCM (Source Control Management)

Understanding SCM is foundational for Git-based workflows in OpenShift and CI/CD pipelines.

### **Core SCM Concepts**

* **Version Control**

  * Track and manage changes in code
* **Branching Strategies**

  * Git Flow, Trunk-Based, Feature Branch
* **Merging & Pull Requests**

  * Collaboration and code review
* **Tags & Releases**

  * Software versioning
* **Repositories (Local & Remote)**

### **Common Git Commands**

```sh
git clone <repo>
git checkout -b feature/login
git add .
git commit -m "Added login module"
git push origin feature/login
git merge feature/login
git tag v1.0
git push --tags
```

### **SCM Providers**

* GitLab
* GitHub
* Bitbucket
* Azure DevOps
* Self-hosted Git solutions

### **Why SCM Matters in OpenShift**

* BuildConfigs and Pipelines depend on Git webhooks
* Automated deployments trigger on repository updates
* Operators, Helm charts, and manifests are managed as code
